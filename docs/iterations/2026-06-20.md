### Iteration 60 Detail — 2026-06-20

**Theme:** Token Refresh / TTL (Issue #32) (Security)

**What We Did:**
Implemented Refresh Token Rotation. Added `RefreshTokens` table in SQLite schema. Updated `POST /api/auth/login` to issue a 7-day refresh token. Created `POST /api/auth/refresh` to exchange valid refresh tokens for new access tokens while instantly invalidating the old refresh token. Documented new endpoints and added integration tests.

**Decisions & Rationales:**
A 24h JWT with a 7-day refresh token rotation ensures secure, long-lived autonomous sessions for the Týr Agent without exposing permanent access tokens.
