# Use size-based log rotation and realistic simulation reset in dry-run

## Context

During the Live Agent MVP testing in Tranche 5, the dry-run simulation mode (`agent start --scenarios ...`) was observed to spam the `data/audit-trail.jsonl` log file, growing it at a rapid and unmanageable rate (1 log per second). This occurred because the dry-run simulation forces an artificial price drift to trigger a rebalance, but the simulated trade execution never updated the `LiveStateManager`. As a result, the portfolio remained breached and continually re-triggered every tick.

Even if fixed, unbounded append-only JSONL files pose a disk exhaustion risk across both test and production environments.

## Options considered

1. **Daily Log Sharding:** (`audit-trail-YYYY-MM-DD.jsonl`) Spreads logs out but does not cap total disk space.
2. **Strict Size-Based Log Rotation:** Cap the file size (e.g., 5MB) and rotate a fixed number of backups (e.g., `.1`, `.2`, `.3`).
3. **No Simulation State Update:** Continue spamming in test environments.
4. **Simulation State Reset:** Intercept the `postTradeSimulation.postTradeState` generated by the `Orchestrator` evaluation and feed it back into the `LiveStateManager` during dry runs.

## Decision

We will implement **Option 2 (Size-Based Rotation)** and **Option 4 (Simulation State Reset)**. 
- `FileAuditStorage` will natively rotate files when they exceed 5MB, maintaining a maximum of 3 historical archives to strictly cap disk usage at 20MB.
- `agent.ts` will wrap the `AuditStorageAdapter` in dry-run mode to intercept triggered trade proposals and apply their `postTradeState` back to the portfolio, completely neutralizing the synthetic log spam.

## Rationale

A strictly bounded log size prevents Out of Memory (OOM) crashes in the Command Center API and prevents disk exhaustion. Applying the simulated trade state back to the portfolio accurately mimics a real broker execution, allowing the continuous loop to verify drift mechanics without producing useless spam.

## Implementation impact

- Added `rotateLogsIfNeeded` to `FileAuditStorage`.
- Modified `agent.ts` to wrap `auditStorage` and conditionally update the `LiveStateManager` with the `postTradeState` when `isTriggered` is true.

## Validation

- Verified unit tests pass.
- Verified dry-run simulation triggers once, then waits until drift is breached again before logging the next proposal.
